registerSingleton : بيعمل واحد بس لما بتطلبه instance
registerLazySingleton : بيعمل instance ويجافظ عليه لحد ما تستخدمه
registerFactory : بيعمل instanceجديد كل مرة بتطلبه

تقسيم ملف DI إلى وحدات modules
 بدل ملف واحد ضخم (>2000 سطر)، نعمل ملفات لكل ميزة أو طبقة:
 core_module.dart
 تسجيل الخدمات الأساسية مثل DioFactory, HiveManager, SharedPreferences.
 data_sources_module.dart
 تسجيل جميع data sources مثل HomeRemoteDataSource, MessagesRemoteDataSource, ProfileRemoteDataSource.
 repositories_module.dart
 تسجيل جميع repositories لكل feature.
 use_cases_module.dart
 تسجيل جميع Use Cases لكل feature.
 blocs_module.dart
 تسجيل جميع BLoCs مع تحديد singleton أو factory لكل واحد.
 feature_modules/
 لكل ميزة رئيسية (مثل Rooms, Messages, Profile, Reels)، نسوي ملف DI خاص بها:
 rooms_module.dart, messages_module.dart, profile_module.dart, reels_module.dart
 main_di.dart
 ملف رئيسي يجمع كل modules مع بعض ويعمل init().
